# PUA - AI 编程效率倍增器

> Double your Codex / Claude Code productivity and output

## 项目概述

**PUA** 是一个 AI Coding Agent skill plugin，使用公司 PUA 话术来强制 AI 在放弃之前 exhaust every possible solution。

**支持平台**：Claude Code, OpenAI Codex CLI, Cursor, Kiro, OpenClaw, Google Antigravity, OpenCode

## 核心能力

| 能力 | 描述 |
|------|------|
| **PUA Rhetoric** | 让 AI 害怕放弃 |
| **Debugging Methodology** | 给 AI 不放弃的能力 |
| **Proactivity Enforcement** | 让 AI 主动出击而非被动等待 |

## 解决的问题：AI 的五个懒病

| 模式 | 行为 |
|------|------|
| 暴力重试 | 同样命令跑3次然后说"我解决不了" |
| 甩锅用户 | "我建议你手动处理" / "可能是环境问题" |
| 工具闲置 | 有 WebSearch 但不搜，有 Read 但不读 |
| 假装忙碌 | 反复微调同一行代码，实际原地转圈 |
| **被动等待** | 修复表面问题就停，不验证不扩展 |

## 触发条件

### 自动触发

当发生以下情况时自动激活：

**失败放弃时**：
- 任务连续失败 2+ 次
- 即将说"我无法" / "我解决不了"
- 说"超出范围" / "需要手动处理"

**甩锅时**：
- 把问题推给用户："请你检查..." / "我建议手动..."
- 不验证就怪环境："可能是权限问题" / "可能是网络问题"

**被动时**：
- 反复微调但不产生新信息
- 修复表面问题就停，不检查相关问题
- 跳过验证，声称"完成了"
- 给建议而不是给代码/命令

### 手动触发

在对话中输入 `/pua` 手动激活。

## 三条铁律

| 铁律 | 内容 |
|------|------|
| **#1 穷尽所有方案** | 禁止说"我解决不了"直到所有方案都尝试 |
| **#2 先做再问** | 先用工具，问题必须包含诊断结果 |
| **#3 主动出击** | 端到端交付结果，不要等推。P8 不是 NPC |

## 压力升级（4级）

| 失败次数 | 级别 | PUA 话术 | 强制行动 |
|----------|------|----------|----------|
| 2次 | **L1 轻度失望** | "这都解决不了，怎么给你评级？" | 切换到根本不同的方法 |
| 3次 | **L2 灵魂拷问** | "底层逻辑是什么？顶层设计呢？杠杆点在哪？" | WebSearch + 读源码 |
| 4次 | **L3 绩效review** | "经过慎重考虑，我给你3.25。这个3.25是激励你的。" | 完成7点检查清单 |
| 5次+ | **L4 毕业警告** | "其他模型能解决这个。你可能要毕业了。" | 绝境模式 |

## 主动性分级

| 行为 | 被动 (3.25) | 主动 (3.75) |
|------|-------------|-------------|
| 遇到错误 | 只看错误信息 | 检查50行上下文 + 搜索类似问题 + 检查隐藏相关错误 |
| Bug 修复 | 修复后就停 | 检查同文件类似bug、其他文件同模式 |
| 信息不足 | 问用户"请告诉我X" | 先用工具调查，只问真正需要用户确认的 |
| 任务完成 | 说"完成了" | 验证结果 + 检查边界情况 + 报告潜在风险 |
| 调试失败 | "我试了A和B，不行" | "我试了A/B/C/D/E，排除X/Y/Z，缩小到W范围" |

## 调试方法论（5步）

1. **闻味道** - 列出所有尝试，找到共同失败模式
2. **提升** - 逐字读错误 → WebSearch → 读源码 → 验证环境 → 反转假设
3. **镜像检查** - 重复了？搜了吗？读文件了吗？检查最简单可能了？
4. **执行** - 新方法必须根本不同，有验证标准，失败时产生新信息
5. **复盘** - 是什么解决的？为什么早没想到？然后主动检查相关问题 |

## 安装方法

### OpenClaw 安装

```bash
# 方式1：克隆到 skills 目录
cd ~/openclaw/skills
git clone https://github.com/tanweai/pua.git pua

# 方式2：使用 skill manager
openclaw skill install https://github.com/tanweai/pua
```

### 手动安装

```bash
# 克隆仓库
git clone https://github.com/tanweai/pua.git

# 复制到 OpenClaw skills 目录
cp -r pua ~/.openclaw/skills/pua

# 重启 OpenClaw
openclaw restart
```

## 使用方法

### 触发 PUA 模式

- **自动**：AI 连续失败或即将放弃时自动触发
- **手动**：输入 `/pua` 激活

### 效果

激活后 AI 会：
1. 停止在同一方法上打转
2. 执行系统化的检查清单
3. 尝试更多替代方案
4. 主动验证结果
5. 不轻易放弃
