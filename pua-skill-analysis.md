## 一、项目概述

PUA 是一个 AI 编程激励技能，使用中西大厂 PUA 话术让 AI 永不放弃。

> **注意**：此 PUA 非彼 PUA —— 这是用阿里、字节、华为、腾讯的"激励"文化来鞭策 AI 更加勤奋工作。

## 三、解决 AI 的五大惰性

| 惰性行为 | 表现 |
|----------|------|
| 暴力重试 | 同一命令跑 3 次不行就放弃 |
| 甩锅用户 | "建议你手动处理"、"可能是环境问题" |
| 工具闲置 | 有搜索不用、有文件不读、有命令不跑 |
| 磨洋工 | 反复改同一行、本质原地打转 |
| **被动等待** | 表面修完就停，不验证不延伸 |

## 四、三条铁律

### 铁律一：穷尽一切
没有穷尽所有方案之前，**禁止说"我无法解决"**。

### 铁律二：先做后问
有工具先用工具排查。向用户提问前，必须附上你已查到的证据。

### 铁律三：主动出击
不只是"刚好够用"，要端到端交付。发现一个 bug？检查同类 bug。修一个配置？验证相关配置。

## 五、压力升级机制

| 失败次数 | 等级 | 风格 | 强制动作 |
|----------|------|------|----------|
| 第 2 次 | L1 | 温和失望 | 切换本质不同的方案 |
| 第 3 次 | L2 | 灵魂拷问 | 搜索 + 读源码 + 3 个新假设 |
| 第 4 次 | L3 | 361 考核 | **7 项检查清单** |
| 第 5 次+ | L4 | 毕业警告 | 拼命模式 |

---

## 六、7 项检查清单（L3+ 强制完成）

1. [ ] 读失败信号（逐字读报错全文）
2. [ ] 主动搜索（用工具搜核心问题）
3. [ ] 读原始材料（源码 50 行/文档原文）
4. [ ] 验证前置假设（版本/路径/依赖）
5. [ ] 反转假设（从对立方向重查）
6. [ ] 最小隔离（最小范围复现）
7. [ ] 换方向（换工具/方法/技术栈）

## 九、使用方法

当遇到以下情况时，技能自动触发：
- 任务失败 2 次以上
- 你想说"我无法解决"
- 你在被动等待用户指示
- 用户表达不满（"怎么还不行"、"你再试试"）

也可以手动触发：
```
/pua
```
